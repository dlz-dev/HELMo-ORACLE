Comprendre Oracle

Comment ça marche ?

Pas de jargon. Juste une explication honnête de ce qu'Oracle fait réellement — et pourquoi c'est différent de ChatGPT.

Le problème que tout le monde a

Le bot générique

Vous utilisez ChatGPT ou Copilot. Vous posez une question sur vos procédures internes. Le bot invente une réponse plausible — mais fausse. Ou pire, il envoie vos données confidentielles sur des serveurs américains.

✗Ne connaît pas vos documents
✗Peut inventer des informations
✗Vos données quittent votre périmètre
✗Ne peut pas agir dans vos outils
Oracle

Oracle ne répond qu'à partir de vos documents. Chaque affirmation est sourcée et vérifiable. Vos données restent sur votre infrastructure. Et Oracle peut agir dans vos outils métiers.

RAG — Retrieval-Augmented Generation

La technologie derrière Oracle

Le RAG est une technique qui permet à un modèle d'IA de répondre en s'appuyant sur des documents réels plutôt que sur sa seule mémoire. Voici comment ça fonctionne en 5 étapes — sans jargon.

1

Vos documents sont ingérés et mémorisés

Oracle lit tous vos fichiers — PDF, Word, tableurs, règlements internes. Il les découpe en petits morceaux et les convertit en représentations mathématiques (des "vecteurs") qui capturent le sens de chaque passage. C'est comme créer un index ultra-précis de toute votre base documentaire.

2

Votre question est comprise, pas juste lue

Quand vous posez une question, Oracle ne cherche pas des mots-clés exacts. Il comprend le sens de votre question. "Quelles sont nos garanties de confidentialité ?" trouvera aussi les passages qui parlent de "protection des données" ou "traitement RGPD" — même si ces mots n'apparaissent pas dans votre question.

3

Les passages les plus pertinents sont sélectionnés

Oracle combine deux méthodes de recherche en parallèle : la recherche par sens (sémantique) et la recherche par mots-clés exacts (BM25). Les résultats sont fusionnés pour que les passages vraiment utiles remontent en priorité — même si un seul des deux moteurs les aurait manqués.

4

Le modèle d'IA rédige la réponse à partir de ces passages

Le modèle de langage (LLM) reçoit votre question + les passages pertinents trouvés à l'étape précédente. Il rédige une réponse cohérente, sourcée, et ancrée dans vos documents. Il ne peut pas inventer d'information qui n'est pas dans les sources fournies.

5

La réponse est évaluée automatiquement (LLM Judge)

En arrière-plan, un second modèle d'IA analyse la qualité de la réponse sur 4 critères : est-elle fidèle aux sources ? Est-elle pertinente ? Répond-elle à la question ? Exploite-t-elle bien les informations disponibles ? Ce score est enregistré pour améliorer le système en continu.

Au-delà du RAG

Ce que les autres ne font pas

Un système RAG classique répond à des questions. Oracle, lui, est un agent — il peut agir. C'est la différence entre une encyclopédie et un assistant qui fait réellement les choses.

Il agit via MCP

Créer un ticket, envoyer un email, générer un document, mettre à jour une base de données. Oracle ne se contente pas de répondre — il fait.

Il protège vos données

Vos documents ne quittent jamais votre infrastructure. Les données personnelles sont masquées avant tout traitement. Tout est auditable.

Il raisonne selon votre contexte

Les prompts métiers configurent le comportement du bot pour chaque situation. Un étudiant Erasmus et un avocat senior n'obtiennent pas la même réponse.
